# 2026世界杯选射手游戏 - 技术设计方案

## 概述
5人参与的蛇形排序选秀游戏，从2026世界杯球队中选择射手。Web应用，通过浏览器（手机/电脑均可）访问，免费部署，支持导出Excel。

---

## 1. 技术选型

| 组件 | 选择 | 原因 |
|---|---|---|
| 后端 | Python 3 + Flask | 简单、免费、易部署 |
| 数据库 | SQLite | 无需数据库服务器，基于文件，零成本 |
| 前端 | Flask模板 + 原生JS + Bootstrap 5 | 微信浏览器兼容，响应式移动端，无需构建步骤 |
| Excel导出 | openpyxl | 服务端生成.xlsx文件 |
| 部署 | Render免费版 / PythonAnywhere | 免费额度足够使用 |

## 3. 蛇形排序逻辑

5名参与者，draft_order 分别为 1-5。

- **奇数轮** (1,3,5,...): 选秀顺序 = [1, 2, 3, 4, 5]
- **偶数轮** (2,4,6,...): 选秀顺序 = [5, 4, 3, 2, 1]

当前轮到谁 = 根据当前轮次和当前pick计算得出。每完成一个pick，pick_number+1，
当pick_number > 5时，进入下一轮，pick_number重置为1。

## 4. API接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | / | 游戏主页面 (HTML) |

| POST | /api/reset | 重置游戏（管理员用） |

## 5. 前端设计 (单页应用)

### 主页面布局 (index.html)

- **顶部区域**: 游戏标题 + 当前轮次/pick显示 + 当前轮到谁（大号醒目文字）
- **球员选择区**: 所有球队以折叠卡片形式展示。每个球员是一个按钮，显示球衣号码+姓名。已被选的球员按钮置灰不可点击。
- **已选名单展示区**: 表格形式展示每位参与者各轮选择的球员。

### 轮次控制方式
- 页面大字体显示"轮到 XXX 选择了"
- 只有轮到的人，球员按钮才可点击
- 其他人可以查看但不能选择
- 前后端都做验证，防止篡改

### 设计要点
- Bootstrap 5 CDN，响应式移动端适配
- 球队用国旗emoji + 队名标题
- 折叠手风琴组件展示球队，节省空间
- 操作成功/失败用Toast通知

### 选人流程详解

#### 模式一：手动选人
1. 页面顶部显示"轮到 XXX 选择"，大号醒目提示
2. 轮到的人，球员按钮高亮可点击；非轮次的人按钮置灰
3. 点击球员按钮 → 弹出确认弹窗（显示球员名 + 号码 + 国旗），**「确认选择」** / **「取消」**
4. 点击「取消」→ 关闭弹窗，按钮保持可点
5. 点击「确认选择」→ 执行选择逻辑：
   - 按钮变为选中状态（绿色边框+禁用）
   - 标记 `turnCompleted = true`，阻止当前回合重复选择
   - 选中记录写入 `mockSelections` 数组
   - 底部「选秀结果汇总」表同步更新
   - 顶端「已选 X 人 / 共 Y 人 · 剩余 Z 人」全局统计同步刷新
   - 界面提示切换为「已选择，即将进入下一顺位...」
   - 1.5秒后自动调用 `advanceTurn()` 进入下一顺位

#### 模式二：预选队列（Preselect）
- 顶部模式切换：**「📋 选人模式」** vs **「📝 预选模式」**
- 预选模式下，所有球员按钮可点击（不论是谁的回合）
- 点击球员 → 加入预选队列，球员角标显示优先级序号
- 队列显示在顶部预选面板，支持以下操作：
  - **移除单个**：每个条目上的 ✕ 按钮
  - **调整顺序**：条目上的 ▲（上移）/ ▼（下移）按钮
  - **清空全部**：面板标题旁的「🗑️ 清空」按钮
- 队列持久化到 `localStorage`（key: `worldcup_preselect_{用户名}`）

#### 模式三：托管模式（Auto-draft / Away Mode）
**应用场景**：参与者有事不在场，提前设定预选队列，轮到其回合时自动选择。

- 开关位于顶部身份区域下方：**「🛌 托管模式」** toggle 开关
- 开启后状态持久化到 `localStorage`（key: `worldcup_autodraft_{用户名}`）

**托管模式的跳转流程：**

```
轮到某参与者 → 检测该人是否开启托管模式
            │
            ├── 未开启 → 正常等待手动选择
            │
            └── 已开启 → 检查其预选队列
                          │
                          ├── 队列为空 → 大屏幕显示醒目提示（所有人可见）：
                          │               ⚠️ XXX 已开启托管，但预选队列为空，无预选球员可选
                          │               同时显示「⏭️ 下一位，继续选秀」按钮
                          │               （任何人可点击，跳过该回合）
                          │
                          └── 队列有球员 → 倒计时1.5秒（autoDraftTimer）
                                             │
                               ┌──────────────┴──────────────┐
                               │                             │
                         倒计时结束                   点击「⚡ 立即选择」
                     （自动触发 doAutoPick()）      （取消倒计时，直接执行）
                               │                             │
                               └──────────────┬──────────────┘
                                              │
                                  遍历预选队列，找到第一个
                                  未被选走的球员
                                              │
                                   ┌──────────┴──────────┐
                                   │                     │
                               找到可用球员           全部已被选走
                                   │                     │
                          1. 标记该球员为已选中       大屏幕醒目提示（所有人可见）：
                          2. 从预选队列移除           ⚠️ XXX 的预选球员都已选走
                          3. 更新全局统计             无预选球员可选
                          4. 更新选秀结果表           同时显示「下一位」按钮
                          5. 更新回合信息            （任何人可点击，跳过该回合）
                          6. 显示「⏭️ 下一位，继续选秀」按钮
                                   │
                             点击「下一位」（任何人可点）
                                   │
                              advanceTurn():
                              1. currentPickIndex++
                              2. 按蛇形规则计算下一顺位参与者
                              3. 更新回合信息显示（轮次/顺位/总计第几选）
                              4. 调用 updateUI() 刷新界面
                              5. 调用 checkAutoDraft()
                                    │
                                    └── 下一位如果也开了托管 → 再次进入此流程
                                        实现连跳（多位缺席时自动连续选人）
```

**「下一位」按钮的可见性：**
- **所有参与者均可看到并点击**「⏭️ 下一位，继续选秀」按钮
- 按钮出现在顶部提示区域 `#turnTip`，全屏可见
- 任何时候出现「下一位」按钮，任何在线的参与者都可以点击推进选秀
- 不需要权限限制，目标是让选秀不卡在任何缺席者那里
- 即使当前轮到的人在场但没操作，其他人也可以点「下一位」跳过

**预选队列已空 / 无预选球员时的界面表现：**
- 在顶部提示区域（`#turnTip`）显示醒目的大号提示文字，**所有人可见**
- 提示内容包含参与者名字和问题描述，不依赖 Toast（Toast 会消失）
- 提示示例：`⚠️ 庆爷 的预选球员都已选走，无预选球员可选`
- 提示持续显示直到有人点击「下一位」推进回合
- 在托管状态区域（`#autoDraftStatus`）同步显示同样信息

#### 轮次推进（Demo 模式）
- 当前原型采用模拟的 `currentPickIndex`（0-based）跟踪选秀进度
- 参与者顺序按蛇形规则计算：
  - 奇数轮：庆爷→耗子→老王→李总→老闫
  - 偶数轮：老闫→李总→老王→耗子→庆爷
- 每完成一个选择，1.5秒后自动调用 `advanceTurn()` 推进到下一顺位（无需点击「下一位」按钮）
- 回合信息（轮次/顺位/总计第几选）动态更新

#### 选秀结果汇总表
- 动态渲染的蛇形表格，行=轮次，列=参与者
- 每一选根据蛇形规则放置到对应参与者的列
- 当前正在选择的单元格高亮显示 **「← 选人中」**
- 支持通过 `renderSummaryTable()` 实时刷新

#### 完整性验证
- 所有选民操作（手动/托管）均调用 `recordSelection()` + `renderSummaryTable()` + `updateGlobalStats()`
- 选择记录写入 `mockSelections` 数组，支持后续导出
- 预选队列中的球员被选中后自动移除

## 11. 关键实现细节

### 服务端选择验证逻辑
```
def validate_selection(participant_name, player_id):
    1. 检查游戏未结束
    2. 检查参与者是否存在
    3. 检查是否轮到该参与者
    4. 检查球员是否存在
    5. 检查球员未被选走
    6. 全部通过则记录选择并推进轮次
```

### 轮次计算逻辑
```
def get_current_turn_info():
    r = current_round  # 当前轮次 1-15
    p = current_pick   # 当前pick 1-5
    
    if p > 5:  # 进入下一轮
        r += 1
        p = 1
    
    if r > 15:  # 游戏结束
        return None
    
    # 奇数轮正序，偶数轮逆序
    order = [1,2,3,4,5] if r % 2 == 1 else [5,4,3,2,1]
    participant_draft_order = order[p - 1]
    return {round: r, pick: p, participant: participant_draft_order}
}
```
